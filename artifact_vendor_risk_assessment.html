Full Vendor Assessments
Cloud Document Storage Provider
SaaS / Storage · Reviewed June
Medium Risk
  • Data accessed: Client PHI-adjacent documents; no direct PHI field storage
  • Security posture: SOC 2 Type II report obtained and reviewed — no material exceptions noted
  • Data handling: Encryption confirmed at rest (AES-256) and in transit (TLS 1.2+) via vendor attestation
  • Contractual: BAA on file; data processing terms reviewed against HIPAA minimum-necessary standard
  • Mitigation: Access limited to named users with MFA enforced; annual re-assessment scheduled
Payroll Processor
Payroll / HR · Reviewed March
High Risk
  • Data accessed: Employee SSNs, direct deposit bank details, and benefits-related PHI
  • Security posture: SOC 1 Type II report reviewed for financial controls; no exceptions affecting data integrity
  • Data handling: Data tokenized for bank routing/account numbers; PHI segmented from general payroll records
  • Contractual: Both BAA (for benefits PHI) and DPA (for general PII) on file and reviewed against current policy
  • Mitigation: High-tier classification triggers semi-annual (not annual) re-screen given breadth of sensitive data
Email & Marketing Platform
Marketing / Comms · Reviewed August
Low Risk
  • Data accessed: Customer/prospect names and email addresses only — no PHI, no financial data
  • Security posture: SOC 2 Type I report reviewed; Type II not yet available from vendor (flagged for follow-up next cycle)
  • Data handling: Standard TLS in transit; vendor confirmed no data resale or third-party sharing in DPA
  • Contractual: DPA on file; no BAA required given no PHI exposure
  • Mitigation: Low-tier classification; annual re-screen sufficient, flagged to request Type II report at renewal
Managed IT Services Provider
IT / Infrastructure · Reviewed May
High Risk
  • Data accessed: Full network and administrative access across all in-scope systems
  • Security posture: SOC 2 Type II report reviewed; one exception noted (patch management SLA missed twice in review period) — followed up directly with vendor, resolved
  • Data handling: Remote access requires MFA and is logged; access reviewed jointly each quarter
  • Contractual: Master Services Agreement includes a security addendum with breach notification terms (72-hour requirement)
  • Mitigation: High-tier classification given breadth of access; quarterly access log review added as compensating control
Background Check Vendor
HR / Screening · Reviewed October
Medium Risk
  • Data accessed: Candidate PII including SSN and criminal history records
  • Security posture: FCRA-compliant attestation on file; SOC 2 report requested but not yet provided by vendor (flagged as open item)
  • Data handling: Vendor confirmed data purged from their systems 90 days post-decision per contract
  • Contractual: DPA on file; FCRA disclosure and authorization process reviewed against current HR intake form
  • Mitigation: Follow-up requested for SOC 2 report before next annual review; interim reliance on FCRA attestation and contractual purge terms
Program note: Two vendors (Payroll Processor, Managed IT Services Provider) are classified High-tier due to breadth/sensitivity of access and moved to a semi-annual or quarterly compensating-control cadence rather than the standard annual re-screen — reflecting a risk-based approach to vendor management rather than a flat one-size-fits-all schedule.
