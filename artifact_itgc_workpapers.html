WP-01 — User Access Provisioning & Deprovisioning
Q1 Testing
Objective: Confirm new access is approved before granted and terminated access is revoked within SLA.
Framework tie: NIST CSF PR.AC-1 / HIPAA §164.308(a)(3)
Population: 42 active accounts across 3 in-scope systems, plus 6 terminations during the period · Sample: 10 new accounts (judgmental, weighted toward privileged access) + all 6 terminations
StepProcedureResult
1Trace 10 sampled new accounts to a signed access request form10/10 traced
2Confirm access level matches documented job role (least privilege)9/10 — 1 exception
3Confirm all 6 terminated accounts revoked within 5-business-day SLA6/6 within SLA
4Confirm quarterly access review sign-off obtained from system ownerEvidence obtained
Exception: One account (contractor-to-employee conversion) retained elevated access beyond documented role scope — logged to CAP tracker as Finding-01.
Conclusion: Control operating effectively with one exception; no control failure.
WP-02 — Change Management
Q2 Testing
Objective: Confirm production changes are documented, reviewed, and approved before deployment.
Framework tie: ISO 27001 A.12.1.2
Population: 24 production changes deployed during the period · Sample: 8 changes (random selection, stratified across all 3 systems)
StepProcedureResult
1Trace each sampled change to a documented change request6/8 — 2 exceptions
2Confirm change request includes reviewer approval prior to deployment date6/8 — 2 exceptions
3Confirm rollback plan documented for high-impact changes3/3 high-impact changes documented
Exception: 2 of 8 sampled changes were deployed directly by the IT Manager without a documented approval trail — both were low-impact configuration changes, but the control gap applies regardless of impact. Logged as Finding-02.
Conclusion: Control gap identified; approval trail requirement not consistently enforced. Remediation required.
WP-03 — Backup & Recovery
Q3 Testing
Objective: Confirm backups run on schedule and restoration is periodically tested.
Framework tie: NIST CSF PR.IP-4 / HIPAA §164.308(a)(7)
Population: Daily backup logs for 3 in-scope systems (365 log entries) · Sample: 1 full quarter of logs (Jul–Sep) + most recent restoration test
StepProcedureResult
1Confirm daily backup job completed successfully across sampled quarter92/92 days — no failures
2Confirm a restoration test was performed within the trailing 12 monthsNo restoration test on record
3Confirm backup retention meets documented policy (90-day minimum)Retention confirmed at 120 days
Exception: No evidence of a restoration test in the trailing 12 months — backups run successfully but recoverability is unverified. Logged as Finding-03 (High priority).
Conclusion: Control gap identified; backup execution is reliable but recovery capability is untested. Remediation required before next audit cycle.
WP-04 — Logical Security / Password Policy
Q3 Testing
Objective: Confirm password and authentication settings meet documented policy across in-scope systems.
Framework tie: ISO 27001 A.9.4.3
Population: 3 in-scope systems · Sample: All 3 (full population — system configuration review, not user sampling)
StepProcedureResult
1Confirm minimum password length (12 characters) enforced by system config3/3 compliant
2Confirm MFA enforced for all remote/privileged access3/3 compliant
3Confirm account lockout after 5 failed login attempts3/3 compliant
Exception: None.
Conclusion: Control operating effectively across all in-scope systems — no exceptions noted.
WP-05 — Segregation of Duties (Accounts Payable / Vendor Setup)
Q4 Testing
Objective: Confirm the individual who sets up a new vendor cannot also approve payments to that vendor.
Framework tie: ISO 27001 A.6.1.2
Population: 14 new vendors onboarded during the period · Sample: All 14 (full population — finance-critical control)
StepProcedureResult
1Identify who created each vendor record in the finance systemDocumented for all 14
2Identify who approved the first payment to each vendorDocumented for all 14
3Confirm no individual performed both roles for the same vendor13/14 — 1 exception
Exception: One vendor was both set up and payment-approved by the Finance Lead due to a coverage gap during a staff absence. Logged as Finding-04.
Conclusion: Control operating effectively overall; isolated exception tied to a staffing gap, not a systemic design failure.
